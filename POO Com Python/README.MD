# Python Orientado a Objetos - Aula 01: A Crise do Software e as Origens da POO

Este repositório contém anotações e códigos práticos baseados na primeira aula do módulo de Programação Orientada a Objetos (POO) do curso de Python, ministrado pelo professor Gustavo Guanabara.
### 📌 Visão Geral 
O objetivo central desta aula é compreender a história da programação e os motivos que levaram ao surgimento do paradigma orientado a objetos. O foco da POO é representar elementos do mundo real dentro de sistemas computacionais.
### 📜 Contexto Histórico e a Crise do Software
A necessidade de novas formas de programar surgiu da Crise do Software da década de 1960. Antes disso, o desenvolvimento passou por várias fases:

    Década de 1940: Programação eletrônica primordial, feita por engenheiros em linguagem de máquina.
    Década de 1950: Surgimento das linguagens de baixo nível (Assembly) e, posteriormente, das linguagens lineares de alto nível (como Fortran, Cobol e C).
    Limitações das Lineares: Utilizavam desvios forçados no código como GOTO e Jump, o que dificultava a manutenção.

### 🐍 Python e a POO
Embora existam discussões sobre a influência de Smalltalk no Python, o criador da linguagem, Guido van Rossum, esclarece que as características de tipagem dinâmica e orientação a objetos do Python foram mais influenciadas pela implementação da linguagem ABC.
#### 🛠️ Conceitos Chave Explorados

    POO (OOP): Programação Orientada a Objetos.
    OOAD: Object Oriented Analysis and Design (Análise e Projeto de Sistemas Orientados a Objetos).
    UML: Unified Modelling Language, um padrão internacional de diagramas para planejar sistemas antes da codificação.
